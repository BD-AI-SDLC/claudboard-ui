claudboard · UX exploration · active run

Active run — orchestration over log

Feedback: the current run screen reads like a tail of a developer log. For PMs, leads and other non-engineering watchers it's noisy and unparseable — too much "what each agent typed", too little "where the work is and what's needed from me". All three proposals keep every piece of information that's there today, but invert the visual hierarchy: pipeline state and run telemetry become the primary signal, and the live stream is treated as a supporting detail rather than the protagonist.

Variant A · Pipeline-first

Telemetry strip on top. Swim-lane below. Log in a drawer.

Reads horizontally, left-to-right, the way an actual build pipeline reads. The 7 phases of the workflow are always visible as a single row across the top of the body — that's the orchestration. Above it sits a row of KPI tiles (progress · elapsed · ETA · cost · gate). Below it sits a "what just happened" storyline rewritten in plain English, with the live stream as a small collapsible drawer beside the telemetry rail.

Emphasises
Where in the 7-phase pipeline the run is, at a glance
Run telemetry as headline numbers, not rail kv-pairs
The single human decision needed, surfaced as a block
Hides without losing
Full stream → bottom drawer (expandable)
Tool calls + agent chatter → tooltip on agent pill
Capability flags → moved into the rail under "Context"

Variant B · Workflow document

Reads like a status report, not a console.

The run is rendered as a structured document — one section per phase, each with a plain-English outcome line, the deliverables, who did it and how long it took. Tabs at the top split the experience into Workflow (default, business-readable), Spec & plan, Artifacts, and Activity (the raw stream for engineers who want it). Right rail is dedicated telemetry — no stream competing for attention.

Emphasises
Phases as the spine of the page, told in business language
Outcome lines, not tool calls — "Drafted 3 scenarios" not "wrote outbox-dispatcher.feature · 142 LOC"
A persistent telemetry rail that summarises the whole run
Hides without losing
Live stream → "Activity" tab (full fidelity, untouched)
Deeper artefacts → "Spec & plan" + "Artifacts" tabs
Per-agent messages → expandable "More" link inside each phase card

Variant C · Cockpit / status board

Headline numbers, a compact pipeline ribbon, a "decision queue".

This direction treats the active run like a status board you'd glance at between meetings. Big KPI tiles dominate. The pipeline is a small horizontal ribbon — present but secondary. The body is a 2-up of "what's happening / what just happened" and a decision queue for gates. The live stream is one of two toggleable feeds on the right ("Activity" vs. "Decisions") so it's there for those who want it, but not in your face.

Emphasises
Glanceable: 4 KPI tiles + 1-line status answer the "where are we?" question instantly
A first-class "Decision queue" — what needs you, in priority order
Toggleable feed: same panel shows either activity or just gates
Hides without losing
Pipeline depth → compact ribbon, expandable from header
Per-agent log → behind feed-toggle (default: Decisions)
Plan + spec → behind "Review" CTA in the decision card
