# Claude API Pricing Reference

Authoritative pricing for Claude models used in the feature workflow.
Source: https://platform.claude.com/docs/en/about-claude/pricing

## Token Pricing (per million tokens)

| Model | Base Input | 5min Cache Write | 1hr Cache Write | Cache Read (Hit) | Output |
|-------|-----------|------------------|-----------------|------------------|--------|
| Claude Opus 4.6 | $5.00 | $6.25 | $10.00 | $0.50 | $25.00 |
| Claude Sonnet 4.6 | $3.00 | $3.75 | $6.00 | $0.30 | $15.00 |
| Claude Haiku 4.5 | $1.00 | $1.25 | $2.00 | $0.10 | $5.00 |

## Cost dimensions explained

- **Base input** — tokens read for the first time, no caching involved
- **5-minute cache write** — tokens written to the prompt cache with default
  5-minute TTL. Costs 1.25x base input. Subsequent reads within the TTL
  are cache hits.
- **1-hour cache write** — tokens written with extended 1-hour TTL. Costs
  2x base input. Used for breakpoint caching and after context compaction.
- **Cache read (hit)** — tokens read from an existing cache entry. Costs
  0.1x base input. The cheapest dimension.
- **Output** — tokens generated by the model. The most expensive dimension
  (5x base input for Sonnet/Haiku, 5x for Opus).

## Cache behavior in Claude Code

- **5-minute TTL (default):** Prompt cache entries last 5 minutes. Sequential
  turns within the same agent hit the cache at 0.1x base input cost.
- **1-hour TTL (extended):** Available for breakpoints; applies after context
  compaction when re-reading large context.
- **Sub-agent spawns:** Each `Agent` tool call creates a FRESH context — no
  cache reuse between spawns. First turn of every spawn pays full cache-write
  price for all input tokens.
- **Context compaction:** When the conversation approaches ~200K tokens, Claude
  Code compresses prior messages. The next turn re-reads the full context,
  paying cache-write price (not cache-hit). This is the primary source of
  cost estimation error — a single compaction can cost as much as several
  normal turns.
- **Multi-turn agents:** Within a single agent spawn, subsequent turns benefit
  from cache hits on the prior context. Only new content (tool results, new
  messages) pays cache-write rates.

## Agent cost profiles (feature workflow)

Each agent in the workflow has a typical token consumption pattern. These
profiles are used for per-agent cost estimation in Phase 7.

**Estimation approach:** For each sub-agent spawn, assume:
- All input on the first turn is priced at **cache-write** rate (fresh context)
- Subsequent turns within the same spawn use **cache-hit** for prior context
  and **cache-write** for new content
- Output tokens are always at output rate
- For simplicity, sub-agent estimates use the cache-write rate for all input
  (conservative upper bound)

| Agent | Model | Typical Spawns | Est. Input/Spawn | Est. Output/Spawn |
|-------|-------|---------------|------------------|-------------------|
| Orchestrator | Sonnet 4.6 | 1 (continuous) | 200K-500K cumulative | 50K-120K cumulative |
| sdd-expert | Sonnet 4.6 | 1 | 25K-40K | 10K-20K |
| architect | Sonnet 4.6 | 1 | 35K-50K | 15K-25K |
| jira-agent | Haiku 4.5 | 3-5 | 5K-10K each | 1K-3K each |
| git-agent | Haiku 4.5 | 3-5 | 5K-10K each | 1K-3K each |
| impl-agent (baseline) | Sonnet 4.6 | 1 | 25K-40K | 5K-15K |
| impl-agent (checkpoint) | Sonnet 4.6 | 1-4 | 80K-200K each | 30K-80K each |
| spec-reviewer | Sonnet 4.6 | 1-2 | 30K-60K | 5K-10K |
| design-reviewer | Sonnet 4.6 | 1-2 | 30K-60K | 5K-10K |
| pr-agent | Sonnet 4.6 | 1 | 10K-20K | 3K-5K |

**Orchestrator note:** The orchestrator is the largest cost driver — it runs
for the entire session, accumulates context across all phases, and may compact
multiple times. Its estimate has the widest error margin. Use a blended input
rate: 30% cache-write + 70% cache-hit, reflecting sequential turns within the
5-minute cache TTL.

## Per-agent cost formula

For sub-agents (fresh context per spawn):
```
cost = (input_tokens x cache_write_rate + output_tokens x output_rate) / 1,000,000
```

For the orchestrator (blended cache):
```
cost = (input_tokens x (0.3 x cache_write_rate + 0.7 x cache_hit_rate)
      + output_tokens x output_rate) / 1,000,000
```

## Example calculations (Sonnet 4.6)

Single sub-agent spawn with 40K input, 15K output:
```
cost = (40,000 x $3.75 + 15,000 x $15.00) / 1,000,000
     = ($150,000 + $225,000) / 1,000,000
     = $0.375
```

Orchestrator with 300K cumulative input, 80K output:
```
blended_input_rate = 0.3 x $3.75 + 0.7 x $0.30 = $1.335
cost = (300,000 x $1.335 + 80,000 x $15.00) / 1,000,000
     = ($400,500 + $1,200,000) / 1,000,000
     = $1.60
```

Haiku sub-agent with 8K input, 2K output:
```
cost = (8,000 x $1.25 + 2,000 x $5.00) / 1,000,000
     = ($10,000 + $10,000) / 1,000,000
     = $0.02
```
